# HeliosEDU — LMS (web-first, аналог Moodle + Прометей)

Next.js (App Router) + TypeScript + Tailwind + SWR + Zustand.
Мокап LMS с трекингом изучения материалов и ролевой моделью. Данные — ПГГПУ/ПНИПУ, факультет ИНЭК, группа М1217.

## Запуск

```bash
npm install
npm run dev
```

http://localhost:3000 · Роль переключается в правом верхнем углу (RoleSwitcher).

## Роли — чем отличаются

| | Студент | Преподаватель | Организатор | Администратор |
|---|---|---|---|---|
| **Кабинет** | Личный | Преподавателя | Организатора | Администрирование |
| **Дашборд** | курсы, дедлайны, прогресс | на проверке, отстающие, прогресс группы | платформа: курсы/юзеры/группы | платформа |
| **Навигация** | Мои курсы, Программы, Хаб, Мои оценки | Мои курсы, **Проверка работ**, **Аналитика**, Мониторинг, Журнал | Курсы, Программы, Аналитика, Журнал + Админ | + Проверка работ, Админ |
| **Журнал** | только свои оценки | таблица группы, **редактирование** | таблица группы (чтение) | редактирование |
| **Хаб группы** | соревнование (лидерборд) | мониторинг (фокус на отстающих) | мониторинг | мониторинг |
| **Статистика курса** | — | ✅ кто как изучил | ✅ | ✅ |
| **Цвет роли** | зелёный | синий | фиолетовый | янтарный |

## Демо-сценарий (для презентации)

1. **Студент (Иван Вагулин, М1217)** — дашборд → «Математический анализ» → **Видеолекция**. Смотри видео: справа панель «Ваша активность» — покрытие растёт в реальном времени, перемотка не засчитывается. Затем **Хаб группы** — ты внизу рейтинга, поднимаешься по мере просмотра. Пройди **Тест** — балл влияет на лидерборд.
2. **Переключи роль → Преподаватель** — другой дашборд (синий), пункты «Проверка работ» и «Аналитика». Открой **Проверка работ** → выбери работу → поставь оценку и комментарий. Зайди в тот же курс → вкладка **Статистика** → раскрой материал → таймлайн действий студента. **Мониторинг группы** — отстающие сверху.
3. **Журнал оценок** под преподавателем — нажми на оценку, измени её (сохраняется). Вернись под студентом — он видит обновлённую оценку в «Мои оценки».
4. **Администратор** — янтарный дашборд со сводкой по платформе, разделы «Пользователи / Группы / Управление курсами».

## Трекинг изучения материалов

- **Видео** (`features/player/VideoPlayer`): покрытие по корзинам 5 сек, гейтинг по фокусу вкладки и скорости, перемотка не засчитывается. ≥90% → зачёт.
- **PDF / презентации** (`DocumentReader`): трекинг просмотренных страниц + время чтения (рендер — заглушка, в проде PDF.js). ≥80% → зачёт.
- **Тесты**: результат идёт в лидерборд и журнал.
- Клиентский трекер: батчинг + `sendBeacon` на закрытие вкладки.
- Серверный мок-store (`mocks/tracking-store`) с предзаполнением группы М1217.

## Структура

```
src/
├── app/(app)/          # авторизованная зона (sidebar+header)
│   ├── dashboard/       # ролевой дашборд (DashboardSwitch)
│   ├── courses/[id]/    # курс (вкладки Содержание/Статистика), video/, document/, quiz/, assignment/
│   ├── submissions/     # проверка работ (tutor/admin)
│   ├── analytics/       # аналитика групп (tutor/org/admin)
│   ├── hub/             # лидерборд / мониторинг
│   ├── gradebook/       # журнал по ролям
│   ├── profile/         # профиль
│   └── admin/           # управление (admin/organizer)
├── app/api/             # мок-бэкенд (route handlers)
├── features/            # dashboard, courses, player, quiz, assignment,
│                        # gradebook, submissions, analytics, hub, programs, notifications
├── shared/              # ui, auth (permissions, role-meta, RoleGate), layout, lib, types
├── stores/              # zustand (session, ui)
└── mocks/               # данные + in-memory stores (tracking, submissions, gradebook)
```

## Стек

Next.js 15 · React 19 · TypeScript · Tailwind · SWR (server state) · Zustand (session/role) · lucide-react · sonner · class-variance-authority.

## Архитектура

- **API-first**: всё через `/api/*`, контракт под реальный бэкенд
- **RBAC**: `permissions.ts` (матрица прав + ролевая навигация), `RoleGate`, `role-meta` (визуальная идентичность)
- **Feature-sliced**: `features/*` изолированы, общение через `shared`
- **Real-time**: SWR `refreshInterval` (мок SSE), polling уведомлений
